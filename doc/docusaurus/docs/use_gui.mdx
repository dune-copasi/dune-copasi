In this setup, DuneCopasi is integrated into the [Spatial Model Editor](/spatial_model_editor), with most
options automatically configured when editing models.

It's important to note that while DuneCopasi's [CLI](use_cli) is designed for configuring parameters
of a generic [mathematical model](/math_model), the Spatial Model Editor focuses on setting up [SBML](http://sbml.org/) models.
In the latter case, the mathematical model is largely inferred from the selected options.

:::tip[SME Documentation]
To learn how to create, edit, and run the models in the Spatial Model Editor please refer to [its documentation](https://spatial-model-editor.readthedocs.io).
:::

### Set Simulation type

To run the Spatial Mode Editor with DuneCopasi, make sure that the simulator is set to **DUNE (FEM)**:

:::info[SME Menu]
**Tools** > **Set simulation type** > **DUNE (FEM)**

### Configuration and grid files

There might be cases where the Spatial Model Editor doesn't represent all of the [configuration options](#configuration-options)
equivalent the to [CLI usage](use_cli) required to run a forward simulation.
In those cases, you can export the configuration file and its associated grid for [CLI usage](use_cli) for the current open model:
